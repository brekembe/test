## Введение

Этот документ представляет собой полную документацию простого компилятора для языка арифметических выражений. Компилятор написан на Python и демонстрирует основные принципы работы компиляторов: лексический анализ, синтаксический анализ, генерацию промежуточного кода и выполнение.

**Поддерживаемые операции:**
- Сложение (`+`)
- Вычитание (`-`)
- Умножение (`*`)
- Деление (`/`)
- Скобки для изменения приоритета операций

**Примеры выражений:**
```
2 + 3
(2 + 3) * 4
((10 - 2) * 3) / 4
```

## Теоретические основы

### Что такое компилятор?

**Компилятор** — это программа, которая преобразует исходный код, написанный на одном языке программирования (языке высокого уровня), в эквивалентный код на другом языке (обычно язык низкого уровня или машинный код).

Основные цели компилятора:
1. **Проверка корректности** — обнаружение синтаксических и семантических ошибок
2. **Преобразование** — перевод программы в форму, удобную для выполнения
3. **Оптимизация** — улучшение производительности конечного кода

### Основные этапы компиляции

Типичный компилятор состоит из следующих этапов:

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  Исходный   │ -> │   Лексер    │ -> │   Парсер    │ -> │     AST     │
│     код     │    │  (Tokens)   │    │             │    │             │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                                        │
                                                        v
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  Результат  │ <- │ Интерпретатор│ <- │ Генератор   │ <- │ Промежуточный│
│  (значение) │    │             │    │    кода     │    │     код     │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

#### 1. Лексический анализ (Lexical Analysis)
Лексер разбивает исходный текст на отдельные токены (лексемы). Токен — это минимальная значимая единица языка (число, оператор, ключевое слово и т.д.).

**Пример:**
```
Исходный код: "2 + 3 * 4"
Токены: [NUMBER(2), PLUS(+), NUMBER(3), MUL(*), NUMBER(4)]
```

#### 2. Синтаксический анализ (Syntax Analysis)
Парсер проверяет, соответствует ли последовательность токенов правилам грамматики языка, и строит абстрактное синтаксическое дерево (AST).

#### 3. Семантический анализ (Semantic Analysis)
Проверка смысловой корректности программы (типы данных, области видимости и т.д.). В нашем простом компиляторе этот этап минимален.

#### 4. Генерация промежуточного кода (Intermediate Code Generation)
Преобразование AST в промежуточное представление, независимое от целевой платформы.

#### 5. Оптимизация (Optimization)
Улучшение промежуточного кода для повышения производительности.

#### 6. Генерация целевого кода (Code Generation)
Преобразование промежуточного кода в машинный код или байт-код.

### Абстрактное синтаксическое дерево (AST)

**AST** — это древовидная структура, представляющая синтаксическую структуру исходного кода. Каждый узел дерева представляет конструкцию языка.

**Пример:**
Для выражения `(2 + 3) * 4` AST будет выглядеть так:

```
       *
      / \
     +   4
    / \
   2   3
```

В нашем компиляторе используются два типа узлов:
- `NumberNode` — листовой узел, содержащий числовое значение
- `BinaryOpNode` — узел бинарной операции с левым операндом, оператором и правым операндом

### Трехадресный код

**Трехадресный код** — форма промежуточного представления, где каждая инструкция содержит не более трёх операндов и одну операцию.

**Формат:**
```
x = y op z
```

где:
- `x`, `y`, `z` — переменные или константы
- `op` — операция (+, -, *, /)

**Пример:**
Для выражения `(2 + 3) * 4`:
```
t0 = 2 + 3
t1 = t0 * 4
ACC = t1
```

Здесь `t0` и `t1` — временные переменные, `ACC` — аккумулятор (регистр для хранения результата).

## Архитектура компилятора

Наш компилятор состоит из следующих основных компонентов:

```
┌─────────────────────────────────────────────────────────────┐
│                        Compiler                             │
│  ┌─────────┐    ┌─────────┐    ┌──────────┐    ┌─────────┐ │
│  │  Lexer  │ -> │ Parser  │ -> │   Code   │ -> │   IR    │ │
│  │         │    │         │    │ Generator│    │         │ │
│  └─────────┘    └─────────┘    └──────────┘    └─────────┘ │
│                                              │              │
│                                              v              │
│  ┌─────────┐    ┌─────────┐                 │              │
│  │ Result  │ <- │Interpreter| <--------------              │
│  └─────────┘    └─────────┘                                 │
└─────────────────────────────────────────────────────────────┘
```

**Классы:**
- `TokenType` — перечисление типов токенов
- `Token` — структура данных для токена
- `Lexer` — лексический анализатор
- `NumberNode`, `BinaryOpNode` — узлы AST
- `Parser` — синтаксический анализатор
- `CodeGenerator` — генератор трехадресного кода
- `Interpreter` — интерпретатор промежуточного кода
- `Compiler` — главный класс, координирующий работу всех компонентов

## Подробное описание компонентов

### Лексер (Lexer)

**Назначение:** Преобразование исходной строки в последовательность токенов.

#### Типы токенов (`TokenType`)

```python
class TokenType(Enum):
    NUMBER   # Число (целое положительное)
    PLUS     # Оператор '+'
    MINUS    # Оператор '-'
    MUL      # Оператор '*'
    DIV      # Оператор '/'
    LPAREN   # Левая скобка '('
    RPAREN   # Правая скобка ')'
    EOF      # Конец входных данных
```

#### Структура токена (`Token`)

```python
@dataclass
class Token:
    type: TokenType      # Тип токена
    value: Union[int, str]  # Значение (число или символ)
```

#### Алгоритм работы лексера

1. **Инициализация:** Сохраняем исходную строку и устанавливаем позицию в 0
2. **Пропуск пробелов:** Игнорируем все пробельные символы
3. **Распознавание токенов:**
   - Если текущий символ цифра → читаем всё число целиком
   - Если символ оператора → создаём соответствующий токен
   - Если скобка → создаём токен скобки
   - Если конец строки → возвращаем токен EOF
   - Иначе → ошибка лексического анализа

#### Методы класса `Lexer`

| Метод | Описание |
|-------|----------|
| `__init__(source)` | Инициализация лексера исходной строкой |
| `peek()` | Возвращает текущий символ без продвижения позиции |
| `advance()` | Возвращает текущий символ и продвигает позицию |
| `skip_whitespace()` | Пропускает все пробельные символы |
| `number()` | Считывает целое число |
| `next_token()` | Возвращает следующий токен |
| `tokenize()` | Возвращает список всех токенов |

#### Пример работы лексера

**Вход:** `"2 + 3 * 4"`

**Процесс:**
1. Пропускаем пробелы (нет)
2. Символ '2' — цифра → вызываем `number()` → токен `NUMBER(2)`
3. Пропускаем пробел → символ '+' → токен `PLUS('+')`
4. Пропускаем пробел → символ '3' — цифра → токен `NUMBER(3)`
5. Пропускаем пробел → символ '*' → токен `MUL('*')`
6. Пропускаем пробел → символ '4' — цифра → токен `NUMBER(4)`
7. Конец строки → токен `EOF("")`

**Выход:** `[NUMBER(2), PLUS('+'), NUMBER(3), MUL('*'), NUMBER(4), EOF("")]`

### Парсер (Parser)

**Назначение:** Преобразование последовательности токенов в абстрактное синтаксическое дерево (AST) согласно правилам грамматики.

#### Грамматика языка

Наш язык описывается следующей контекстно-свободной грамматикой:

```
expression := term (('+' | '-') term)*
term       := factor (('*' | '/') factor)*
factor     := NUMBER | '(' expression ')'
```

Эта грамматика определяет:
- **Приоритет операций:** Умножение и деление имеют более высокий приоритет, чем сложение и вычитание
- **Ассоциативность:** Все операции левоассоциативны
- **Скобки:** Позволяют изменять стандартный порядок вычислений

#### Узлы AST

**NumberNode** — представляет числовое значение:
```python
@dataclass
class NumberNode:
    value: int  # Числовое значение
```

**BinaryOpNode** — представляет бинарную операцию:
```python
@dataclass
class BinaryOpNode:
    left: ASTNode      # Левый операнд
    op: TokenType      # Оператор (+, -, *, /)
    right: ASTNode     # Правый операнд
```

#### Алгоритм парсинга (рекурсивный спуск)

Метод рекурсивного спуска — это техника нисходящего парсинга, при которой для каждого правила грамматики создаётся функция.

**Структура методов:**

1. **`expression()`** — обрабатывает сложение и вычитание
   - Вызывает `term()` для получения первого операнда
   - Пока текущий токен '+' или '-':
     - Запоминает оператор
     - Вызывает `term()` для правого операнда
     - Создаёт узел `BinaryOpNode`
   - Возвращает построенное дерево

2. **`term()`** — обрабатывает умножение и деление
   - Вызывает `factor()` для получения первого операнда
   - Пока текущий токен '*' или '/':
     - Запоминает оператор
     - Вызывает `factor()` для правого операнда
     - Создаёт узел `BinaryOpNode`
   - Возвращает построенное дерево

3. **`factor()`** — обрабатывает числа и скобки
   - Если токен NUMBER → возвращает `NumberNode`
   - Если токен LPAREN → 
     - Потребляет '('
     - Рекурсивно вызывает `expression()`
     - Потребляет ')'
     - Возвращает результат
   - Иначе → ошибка парсинга

#### Методы класса `Parser`

| Метод | Описание |
|-------|----------|
| `__init__(tokens)` | Инициализация списком токенов |
| `current()` | Возвращает текущий токен |
| `consume(expected)` | Проверяет и потребляет ожидаемый токен |
| `parse()` | Запускает парсинг и возвращает AST |
| `expression()` | Парсит выражение (сложение/вычитание) |
| `term()` | Парсит терм (умножение/деление) |
| `factor()` | Парсит фактор (число или скобки) |

#### Пример построения AST

**Вход:** `"(2 + 3) * 4"`

**Токены:** `[LPAREN, NUMBER(2), PLUS, NUMBER(3), RPAREN, MUL, NUMBER(4), EOF]`

**Процесс:**
1. `parse()` вызывает `expression()`
2. `expression()` вызывает `term()`
3. `term()` вызывает `factor()`
4. `factor()` видит LPAREN:
   - Потребляет '('
   - Вызывает `expression()` внутри скобок:
     - `expression()` вызывает `term()` → `factor()` → `NumberNode(2)`
     - Видит PLUS, запоминает оператор
     - Вызывает `term()` → `factor()` → `NumberNode(3)`
     - Создаёт `BinaryOpNode(NumberNode(2), PLUS, NumberNode(3))`
   - Потребляет ')'
   - Возвращает узел сложения
5. `term()` видит MUL:
   - Запоминает оператор
   - Вызывает `factor()` → `NumberNode(4)`
   - Создаёт `BinaryOpNode(узел_сложения, MUL, NumberNode(4))`
6. Возвращает итоговое дерево

**Результат (AST):**
```
       *
      / \
     +   4
    / \
   2   3
```

### Генератор кода (Code Generator)

**Назначение:** Преобразование AST в трехадресный промежуточный код.

#### Алгоритм генерации кода

Генерация происходит методом обхода дерева в глубину (post-order traversal):

1. **Для NumberNode:** Просто возвращаем строковое представление числа
2. **Для BinaryOpNode:**
   - Рекурсивно генерируем код для левого операнда → получаем имя переменной или значение
   - Рекурсивно генерируем код для правого операнда → получаем имя переменной или значение
   - Создаём новую временную переменную
   - Генерируем инструкцию: `temp = left op right`
   - Возвращаем имя временной переменной

3. **После обработки корня AST:** Добавляем инструкцию сохранения результата в аккумулятор: `ACC = result`

#### Методы класса `CodeGenerator`

| Метод | Описание |
|-------|----------|
| `__init__()` | Инициализация списка инструкций и счётчика временных переменных |
| `new_temp()` | Создаёт имя новой временной переменной (t0, t1, t2, ...) |
| `generate(node)` | Генерирует полный код для AST, возвращает строку |
| `emit(node)` | Рекурсивно обрабатывает узел и возвращает его «имя» (значение или temp) |

#### Пример генерации кода

**Вход (AST):** Дерево для `(2 + 3) * 4`

**Процесс:**
1. Вызываем `generate()` для корневого узла (*)
2. `emit()` для *:
   - Вызывает `emit()` для левого операнда (+):
     - Вызывает `emit()` для 2 → возвращает "2"
     - Вызывает `emit()` для 3 → возвращает "3"
     - Создаёт temp `t0`
     - Добавляет инструкцию: `t0 = 2 + 3`
     - Возвращает "t0"
   - Вызывает `emit()` для правого операнда (4) → возвращает "4"
   - Создаёт temp `t1`
   - Добавляет инструкцию: `t1 = t0 * 4`
   - Возвращает "t1"
3. Добавляем финальную инструкцию: `ACC = t1`

**Выход (промежуточный код):**
```
t0 = 2 + 3
t1 = t0 * 4
ACC = t1
```

### Интерпретатор (Interpreter)

**Назначение:** Выполнение сгенерированного промежуточного кода.

#### Алгоритм интерпретации

1. Разбиваем код на строки
2. Для каждой строки:
   - Пропускаем пустые строки
   - Разбираем инструкцию присваивания: `dest = expr`
   - Вычисляем выражение справа от '=' с помощью встроенной функции `eval()`
   - Сохраняем результат:
     - Если назначение в `ACC` → сохраняем в переменную аккумулятора
     - Иначе → сохраняем во временную переменную
3. Возвращаем значение аккумулятора

#### Методы класса `Interpreter`

| Метод | Описание |
|-------|----------|
| `__init__()` | Инициализация словаря переменных |
| `interpret(code)` | Выполняет код и возвращает результат |

#### Пример выполнения

**Вход (код):**
```
t0 = 2 + 3
t1 = t0 * 4
ACC = t1
```

**Процесс:**
1. Строка `t0 = 2 + 3`:
   - Вычисляем `2 + 3` → 5
   - Сохраняем: `variables['t0'] = 5`
2. Строка `t1 = t0 * 4`:
   - Вычисляем `t0 * 4` → `5 * 4` → 20
   - Сохраняем: `variables['t1'] = 20`
3. Строка `ACC = t1`:
   - Вычисляем `t1` → 20
   - Сохраняем: `acc = 20`
4. Возвращаем `acc` → 20

**Результат:** `20`

## Пример работы компилятора

Рассмотрим полный цикл компиляции для выражения: `((2 + 3) * 4 - 10) / 2`

### Шаг 1: Лексический анализ

**Вход:** `"((2 + 3) * 4 - 10) / 2"`

**Токены:**
```
[
  LPAREN('('),
  LPAREN('('),
  NUMBER(2),
  PLUS('+'),
  NUMBER(3),
  RPAREN(')'),
  MUL('*'),
  NUMBER(4),
  MINUS('-'),
  NUMBER(10),
  RPAREN(')'),
  DIV('/'),
  NUMBER(2),
  EOF('')
]
```

### Шаг 2: Синтаксический анализ

**Построение AST:**

```
          /
        /   \
       -     2
     /   \
    *     10
   / \
  +   4
 / \
2   3
```

### Шаг 3: Генерация кода

**Промежуточный код:**
```
t0 = 2 + 3
t1 = t0 * 4
t2 = t1 - 10
t3 = t2 / 2
ACC = t3
```

### Шаг 4: Интерпретация

**Выполнение по шагам:**
1. `t0 = 2 + 3` → t0 = 5
2. `t1 = t0 * 4` → t1 = 20
3. `t2 = t1 - 10` → t2 = 10
4. `t3 = t2 / 2` → t3 = 5
5. `ACC = t3` → ACC = 5

**Результат:** `5`

### Проверка

```python
>>> eval("((2 + 3) * 4 - 10) / 2")
5.0
```

✓ Результат совпадает!

## Запуск и использование

### Требования

- Python 3.7 или выше
- Никаких внешних зависимостей не требуется

### Запуск компилятора

```bash
python3 simple_compiler.py
```

### Режимы работы

#### 1. Автоматический режим (демонстрация)

При запуске без параметров компилятор выполняет набор тестовых выражений:

```
==================================================
ПРОСТОЙ КОМПИЛЯТОР АРИФМЕТИЧЕСКИХ ВЫРАЖЕНИЙ
==================================================

Выражение: 2 + 3
Сгенерированный код:
  t0 = 2 + 3
  ACC = t0
Результат: 5
Ожидается: 5 ✓

Выражение: (2 + 3) * 4
Сгенерированный код:
  t0 = 2 + 3
  t1 = t0 * 4
  ACC = t1
Результат: 20
Ожидается: 20 ✓

...
```

#### 2. Интерактивный режим

После демонстрации тестов компилятор переходит в интерактивный режим:

```
==================================================
Интерактивный режим (введите 'quit' для выхода)

def number(self) -> Token:
    start = self.pos
    while self.peek() and (self.peek().isdigit() or self.peek() == '.'):
        self.advance()
    value = float(self.source[start:self.pos])
    return Token(TokenType.NUMBER, value)
```

def optimize_constant_folding(self, node: ASTNode) -> ASTNode:
    if isinstance(node, BinaryOpNode):
        left = self.optimize_constant_folding(node.left)
        right = self.optimize_constant_folding(node.right)
        
        if isinstance(left, NumberNode) and isinstance(right, NumberNode):
            # Вычисляем результат на этапе компиляции
            result = self.evaluate(left.value, node.op, right.value)
            return NumberNode(result)
        
        return BinaryOpNode(left, node.op, right)
    
    return node

### 6. Генерация кода для другой целевой платформы

- JavaScript код

Пример генерации JavaScript:

```python
def generate_js(self, node: ASTNode) -> str:
    if isinstance(node, NumberNode):
        return str(node.value)
    
    if isinstance(node, BinaryOpNode):
        left = self.generate_js(node.left)
        right = self.generate_js(node.right)
        op = self.op_map[node.op]
        return f"({left} {op} {right})"
```

## Заключение

Этот простой компилятор демонстрирует фундаментальные принципы работы компиляторов:

1. **Модульность** — каждый компонент (лексер, парсер, генератор кода) выполняет свою задачу
2. **Разделение ответственности** — чёткое разграничение между этапами компиляции
3. **Промежуточное представление** — использование трехадресного кода как универсального формата
4. **Обработка ошибок** — информативные сообщения об ошибках на каждом этапе

Несмотря на простоту, этот компилятор реализует все основные концепции, используемые в промышленных компиляторах:

- Контекстно-свободные грамматики
- Рекурсивный спуск
- Абстрактные синтаксические деревья
- Трехадресный код
- Интерпретация промежуточного представления

Изучив этот код, вы получите прочную основу для понимания более сложных систем компиляции, таких как GCC, LLVM, V8 JavaScript Engine и других.
